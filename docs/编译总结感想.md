# 编译总结感想

22377311 王岱泽

​	这学期的编译课是我感觉在我目前的本科生涯里任务量最大的一门课了。自己一个人开发一个10000行的项目，并且对其正确性要求较高，对项目理解要求高，这是我目前自己一个人做过最大的项目，也算是有所成就，有所收获。

​	我在开始词法语法语义分析的时候感觉还较为轻松，当时是一次次把编译理论课的内容用于实践，确实要说，编译的理论和实践结合的很紧密、上课学的实验真能用到。

​	但是到了中间代码生成，我感觉难度一下就不一样了，因为是把自己的语法树要转化为一个自己之前完全不了解的LLVM中间代码结构，这里尤其是在LLVM框架的搭建上，我花了很多时间，这里思考的过程也很痛苦，最后才理解了LLVM的类型系统、常量系统、皆Value的思想。语法树到LLVM的转化就没有搭建LLVM的框架那么困难了。

​	目标代码生成也是一样，但是比中间代码LLVM的框架要好理解些，所以搭建MIPS框架并没有花费我很多时间，相反，LLVM转MIPS最难的我认为就是对于变量在栈和寄存器的处理和函数调用的填参这两步，因为这里很容易出错，尤其是函数调用。为了简单，我先用的全部栈式处理，像PCODE一样，但是这里对于SP的处理很麻烦，于是我采用了FP寄存器。

​	然后优化又是很难的一步，我参考了虎书学习了它的图着色分配寄存器，这一步要修改很多已经在我的栈式寄存器分配中的内容，但是这是值得的，图着色的优化效果很明显。另外我感觉很值得的优化就是mem2Reg，它的效果也很显著，这里我感觉消除phi指令也是不简单的一步，这里也因为自己之前不好的设计导致出现了不少错误，但是思考之后我把之前的不好的设计删掉，像空行mipsEmpty这种视为一个MIPS指令的东西（其实当初是为了自己方便debug生成的MIPS代码加进去的）。

​	我认为这个编译器的最核心点就在于我建立的那棵语法树，它从语法分析一路走到了LLVM生成，生成LLVM树，再生成MIPS树，建立一课好的语法树是关键的。

​	总之，这是一次很艰巨的任务，但是在引导之下，也是一步步走过来了，自己一个人也是做出了这么一个巨大的项目，是一个对于理论的实践。也让我对于自己写代码有了更深刻的思考，像怎么写代码让编译器更好处理等等。同时我也使用到了大模型来进行debug，有的时候它能找到错误，但是需要我层层引导，但是也有时候是找不到错误，还是要靠我去把错误一行一行找出来，对于编译器这种逻辑复杂且精细的代码，不能对其产生依赖，程序员还是要对代码有足够的了解。

​	最后我要感谢老师和助教的帮助与教导，感谢我的舍友帮助我debug和交流思路，感谢讨论区的同学们的互帮互助，你们帮助我走过了编译的路。
